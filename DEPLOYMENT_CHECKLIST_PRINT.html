First Deployment Checklist

Use this sheet when moving Trimax from the local computer to GitHub, Vercel, and Supabase for the first live deployment.

Workspace R&L Creations + Just Kleen
Date May 27, 2026
Goal Launch-ready Vercel site

1. Local Computer Check

  • Open VS Code in C:\Users\robbi\trimax.
  • Run the app check in the VS Code terminal:
npm run check
  • Confirm lint passes.
  • Confirm the production build passes.
  • Confirm there are no unfinished files in Git before pushing.

2. Supabase Database Check

  • Confirm Supabase Auth works for Robbie and Lyubov.
  • Confirm business_users gives Robbie access to R&L Creations.
  • Confirm business_users gives Lyubov access to Just Kleen.
  • Confirm business_id is populated on business-owned records.
  • Confirm property_users setup is ready for property portal users.
  • Confirm activity_logs setup is ready.
  • Confirm invoice updated_at setup has run.
  • Confirm queue renovation fields have run.
Supabase may warn that some setup scripts include destructive operations. For the current Trimax setup scripts, this is usually because policies or triggers are being recreated. It should not delete invoices, clients, queue items, or business data.

3. GitHub Push

  • Confirm the latest code is committed.
  • Push the latest main branch to GitHub.
  • Confirm GitHub shows the newest commit.
git status git push origin main

4. Create Vercel Project

  • Open Vercel.
  • Choose New Project.
  • Import the Trimax GitHub repository.
  • Keep framework as Next.js.
  • Add the required environment variables.
NEXT_PUBLIC_SUPABASE_URL NEXT_PUBLIC_SUPABASE_ANON_KEY
  • Deploy the project.
  • Copy the live Vercel URL.

5. Supabase Auth URL Settings

  • Open Supabase Authentication URL Configuration.
  • Add the live Vercel URL as the Site URL.
  • Add allowed redirect URLs for login and reset password.
https://YOUR-VERCEL-SITE.vercel.app https://YOUR-VERCEL-SITE.vercel.app/reset-password

6. R&L Creations Smoke Test

  • Open /?business=rnl-creations.
  • Login as Robbie.
  • Open Dashboard, Queue, Schedule, Estimates, Invoices, Payments, Reports, Activity, Settings.
  • Create a test queue item.
  • Confirm renovation fields save correctly if used.
  • Schedule a queue item.
  • Confirm Add To Calendar downloads an .ics file.
  • Create or open an estimate.
  • Create or open an invoice.
  • Confirm new invoice numbers start at or above INV-0500.
  • Confirm new estimate numbers start at or above EST-0500.
  • Print an invoice page.
  • Confirm Activity shows important actions.

7. Just Kleen Smoke Test

  • Open /?business=just-kleen.
  • Login as Lyubov.
  • Open Dashboard, Clients, Services, Estimates, Invoices, Settings.
  • Create or open a test invoice.
  • Confirm Just Kleen wording is correct.
  • Confirm the special 5 Star / BOA print/export workflow still appears when appropriate.

8. Phone App Check

  • Open Settings, then Phone App + Alerts.
  • On iPhone, open the live Vercel site in Safari.
  • Tap Share.
  • Choose Add to Home Screen.
  • Open Trimax from the new Home Screen icon.
Push notifications are not active yet. The app install foundation is ready; notification signup and sending will be added after deployment.

9. Do Not Break During Launch

  • Preserve ?business=rnl-creations.
  • Preserve ?business=just-kleen.
  • Preserve business-specific filtering.
  • Preserve queue bypass for normal estimates/invoices.
  • Preserve apartment split invoice workflow.
  • Preserve Just Kleen special invoice workflow.
  • Preserve printable invoice and estimate pages.
  • Preserve property portal access boundaries.

10. After Launch

  • Add real push notifications for property queue submissions.
  • Improve phone calendar integration.
  • Finish Microsoft Outlook draft integration.
  • Tighten production RLS before selling Trimax as SaaS.
  • Add photo attachments and stronger property manager portal controls.
Development-friendly RLS is acceptable while testing with trusted users, but production RLS must be tightened before Trimax is offered as a paid SaaS product.
